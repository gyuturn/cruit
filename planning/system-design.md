# 취업 공고 추천 AI 시스템 설계 문서

## 1. 프로젝트 개요

사용자의 프로필 정보를 기반으로 맞춤형 취업 공고를 추천하는 AI 시스템

## 2. 요구사항 분석

### 2.1 기능적 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| FR-01 | 사용자 정보 입력 | 신입/경력, 학력, 전공, 자격증 입력 |
| FR-02 | 정보 저장 | 최초 입력 시 파일로 저장, 이후 재사용 |
| FR-03 | 초기화 기능 | 저장된 정보 삭제 및 새로운 입력 가능 |
| FR-04 | 공고 링크 제공 | 취업 공고 원본 링크 제공 |
| FR-05 | 공고 정보 요약 | 공고의 간략한 정보 표시 |
| FR-06 | 웹 크롤링 | 외부 취업 사이트에서 공고 데이터 수집 |
| FR-07 | AI 추천 | 사용자 프로필 기반 맞춤 추천 |
| FR-08 | 페이지네이션 | 초기 5개 표시, 더보기로 추가 조회 |
| FR-09 | 갱신 기능 | 새로운 추천 결과 조회 |
| FR-10 | 경력사항 입력 | 경력자 선택 시 경력 내용/기간 입력 (선택) |
| FR-11 | 추천 평점 | AI 추천 공고에 대해 별점(1-5) 평가 |

### 2.2 사용자 입력 정보

```
- 신입/경력 여부: enum (신입, 경력)
- 대학교 4년제 여부: boolean
- 전공: string
- 자격증 리스트: string[]
- 경력사항 (경력 선택 시, 선택):
  - 회사명: string
  - 직무/직책: string
  - 시작일: YYYY-MM
  - 종료일: YYYY-MM 또는 '재직중'
  - 업무 내용: string (선택)
```

### 3.2 기술 스택 (제안)

| 영역 | 기술 | 선택 이유 |
|---|---|---|
| Frontend | React + TypeScript | 컴포넌트 기반 UI, 타입 안정성 |
| Backend | Node.js + Express | 빠른 개발, JavaScript 통일 |
| 크롤링 | Puppeteer / Cheerio | 동적/정적 페이지 대응 |
| AI 추천 | OpenAI API / 자체 모델 | 자연어 기반 매칭 |
| 저장소 | JSON 파일 / SQLite | 간단한 데이터 구조 |

## 4. 데이터 모델

### 4.1 경력사항 (CareerHistory)

```typescript
interface CareerHistory {
  company: string;      // 회사명
  position: string;     // 직무/직책
  startDate: string;    // 시작일 (YYYY-MM)
  endDate: string;      // 종료일 (YYYY-MM) 또는 '재직중'
  description?: string; // 업무 내용 (선택)
}
```

### 4.2 사용자 프로필 (UserProfile)

```typescript
interface UserProfile {
  id: string;
  experienceLevel: 'junior' | 'experienced';  // 신입/경력
  isFourYearUniv: boolean;                     // 4년제 여부
  major: string;                                // 전공
  certifications: string[];                     // 자격증 리스트
  careerHistory?: CareerHistory[];              // 경력사항 (경력자만)
  createdAt: Date;
  updatedAt: Date;
}
```

### 4.3 취업 공고 (JobPosting)

```typescript
interface JobPosting {
  id: string;
  title: string;           // 공고 제목
  company: string;         // 회사명
  location: string;        // 근무지
  experienceLevel: string; // 경력 조건
  education: string;       // 학력 조건
  skills: string[];        // 필요 기술/자격증
  salary: string;          // 급여 정보
  deadline: Date;          // 마감일
  url: string;             // 원본 링크
  source: string;          // 크롤링 출처
  summary: string;         // 간략 요약
  createdAt: Date;
}
```

### 4.4 추천 결과 (Recommendation)

```typescript
interface Recommendation {
  jobPosting: JobPosting;
  matchScore: number;      // 매칭 점수 (0-100)
  matchReasons: string[];  // 매칭 이유
  userRating?: number;     // 사용자 평점 (1-5)
}
```

### 4.5 공고 평점 (JobRating)

```typescript
interface JobRating {
  jobId: string;           // 공고 ID
  rating: number;          // 별점 (1-5)
  createdAt: string;       // 평가 시간
}
```

## 5. API 설계

### 5.1 사용자 프로필 API

| Method | Endpoint | 설명 |
|---|---|---|
| GET | /api/profile | 저장된 프로필 조회 |
| POST | /api/profile | 새 프로필 저장 |
| DELETE | /api/profile | 프로필 초기화 |

### 5.2 공고 추천 API

| Method | Endpoint | 설명 |
|---|---|---|
| GET | /api/recommendations | 추천 공고 조회 (페이지네이션) |
| POST | /api/recommendations/refresh | 새로운 추천 갱신 |

### 5.3 크롤링 API

| Method | Endpoint | 설명 |
|---|---|---|
| POST | /api/crawl | 크롤링 실행 (내부용) |

## 6. 핵심 기능 상세

### 6.1 크롤링 대상 사이트 (예시)

- 사람인 (saramin.co.kr)
- 잡코리아 (jobkorea.co.kr)
- 원티드 (wanted.co.kr)
- 인크루트 (incruit.com)

> ⚠️ 크롤링 시 각 사이트의 robots.txt 및 이용약관 준수 필요

### 6.2 AI 추천 알고리즘 흐름

```
1. 사용자 프로필 로드
2. 크롤링된 공고 데이터 로드
3. 각 공고에 대해 매칭 점수 계산
   - 경력 조건 일치도
   - 학력 조건 일치도
   - 전공 관련도
   - 자격증 일치도
4. 점수 기반 정렬
5. 상위 N개 반환
```

### 6.3 파일 저장 구조

```
/data
  /profiles
    └── user-profile.json    # 사용자 프로필
  /jobs
    └── jobs-{date}.json     # 크롤링된 공고 데이터
  /cache
    └── recommendations.json # 추천 캐시
```

## 9. 고려사항

### 9.1 법적 고려사항
- 크롤링 대상 사이트의 이용약관 확인
- 개인정보 처리 최소화
- robots.txt 준수

### 9.2 기술적 고려사항
- 크롤링 요청 간격 조절 (Rate Limiting)
- 크롤링 데이터 캐싱
- 에러 처리 및 재시도 로직

### 9.3 확장성
- 추가 취업 사이트 연동 용이한 구조
- 추천 알고리즘 교체 가능한 구조
